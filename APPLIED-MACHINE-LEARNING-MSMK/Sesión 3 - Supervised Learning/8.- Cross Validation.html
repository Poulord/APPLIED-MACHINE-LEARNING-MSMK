Validación Cruzada: La Clave para el Trade-off

Un método esencial para evaluar la capacidad de generalización de un modelo y seleccionar sus hiperparámetros.

El Método K-Fold: 5 Pasos Clave

En lugar de una única división (Entrenamiento/Prueba), dividimos el conjunto de entrenamiento en K subconjuntos iguales (folds). Usaremos K=5 como ejemplo.

DATO TOTAL:
Datos de Entrenamiento (80%)
Datos de Prueba (20%)
PROCESO K=5:
Iteración 1:
Fold 1 (Val)
Folds 2-5 (Train)
(Error E1)
Iteración 2:
Fold 2 (Val)
(Error E2)
... Repetido 5 veces ...
RESULTADO:

El Error de Validación Cruzada es el promedio de los K errores: $$E_{CV} = \frac{1}{K} \sum_{i=1}^{K} E_i$$

El Punto Ideal (Low Error)

La curva de la Validación Cruzada (línea roja) nos guía.

  • El error más bajo en la línea roja (pico verde) indica la Complejidad Óptima.
  • A este nivel, la Varianza es controlada y el Sesgo es reducido.

Resultado:

La validación cruzada evita usar el conjunto de prueba (test) para ajustar el modelo, asegurando una evaluación final imparcial.

Recuerda: La validación cruzada se ejecuta en los datos de entrenamiento para elegir los hiperparámetros. El conjunto de prueba final nunca se toca hasta el final.
